# Codex 5.6 — Phase B (read-only review subsystem + capability map) Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax. Pure-unit tasks marked **Dog-food candidate** are handed to `codex-delegate` itself.

**Goal:** Add three read-only "judge" capabilities — `code-review`, `audit`, `plan-review` — that Codex (sol by default, terra for routine diffs) runs as an advisory second opinion, plus rewrite `SKILL.md` into a capability map so Claude knows the full arsenal at planning time.

**Architecture:** Reuse the execution infra (`Runner`, `Executor`, `resolveCodexRuntime`, fallback `nextAction`/`classifyFailure`, `Ledger`). Reviews write nothing, so they bypass the whitelist / clean-tree / verifier / snapshot machinery entirely — a distinct, lighter path. `code-review` wraps the **native `codex review`** (model set via `-c model=`, output = its stdout); `audit`/`plan-review` use `codex exec --sandbox read-only` with a purpose-built prompt (output = last-message file). Findings are returned as **raw text** (advisory) — Claude reads and critically verifies them; no fragile machine-parsing.

**Tech Stack:** Node ≥18.18, TypeScript strict, vitest, smol-toml. Windows-first. Requires Codex CLI ≥ 0.144.1.

**Spec:** `docs/superpowers/specs/2026-07-10-codex-5.6-models-and-review-design.md` (§5, §6, §9.2).

**Grounding facts established empirically (2026-07-10):**
- `codex review` has **no `-m`**; the model is set with `-c model="..."` — verified: a
  `codex review --commit HEAD -c model="gpt-5.6-terra"` run reported `model: gpt-5.6-terra`.
- Native review output is **prose on stdout**, ending in a `codex` summary line then
  `Review comment:` bullets like `- [P3] <title> — <file:line>`. Not JSON → we return
  the raw text, not a parsed structure.
- Native review defaults to `sandbox: workspace-write`; our custom audit/plan path uses
  `--sandbox read-only` (a valid `codex exec -s` value).
- Real code refs: `Executor.run(ExecRequest)` (`src/executor.ts`) builds args via
  `buildCodexArgs` + `--output-last-message` + stdin prompt + `resolveCodexRuntime(auth)`.
  `Controller.delegate` (`src/controller.ts`) is the fallback-ladder template.
  `SANDBOX_LEVELS`/`SandboxLevel` at `src/config/types.ts:19-21`; `sandboxFlags` switch at
  `src/exec/codexArgs.ts:23-46`; `dispatch`/`CliHandlers` at `src/cli.ts:47-80`.

export function validateReviewSpec(
  spec: Partial<ReviewSpec>,
): asserts spec is ReviewSpec {
  if (!spec.repoPath || !isAbsolute(spec.repoPath))
    throw new Error('spec.repoPath must be an absolute path');
  if (!spec.reviewId) throw new Error('spec.reviewId is required');
  if (!spec.reviewType || !REVIEW_TYPES.includes(spec.reviewType))
    throw new Error(`spec.reviewType must be one of ${REVIEW_TYPES.join(', ')}`);
  if (!spec.target) throw new Error('spec.target is required');
  if (spec.auth !== undefined && !AUTH_MODES.includes(spec.auth))
    throw new Error(`spec.auth must be one of ${AUTH_MODES.join(', ')}`);
}
```
(There is NO whitelist check — reviews write nothing, so the primary write-guard
does not apply. This is the intended contract difference from
`validateDelegationSpec`.)

import type { ReviewSpec } from './config/types.js';

const FORMAT =
  'Report findings as a list, each: `severity (P0-P3) | file:line | claim | why`. ' +
  'Be specific and cite exact paths/lines. This is a READ-ONLY review: do not ' +
  'modify any file. If nothing is wrong, say so plainly.';

export function buildReviewPrompt(spec: ReviewSpec): string {
  switch (spec.reviewType) {
    case 'audit':
      return `# Audit ${spec.reviewId}\n\nAudit the code under \`${spec.target}\`` +
        (spec.focus ? ` with a focus on ${spec.focus}` : '') +
        `.\nLook for correctness, security, and quality issues.\n\n${FORMAT}`;
    case 'plan-review':
      return `# Plan review ${spec.reviewId}\n\nRead the plan/spec at ` +
        `\`${spec.target}\` and critique it BEFORE it is implemented: gaps, wrong ` +
        `assumptions, risky steps, missing edge cases, anything that would cause ` +
        `rework.\n\n${FORMAT}`;
    case 'code-review':
      throw new Error('code-review uses the native `codex review`, not a built prompt');
  }
}

import type { Effort } from '../config/types.js';

export interface ReviewInvocation {
  readonly target: string;
  readonly model: string;
  readonly effort: Effort;
}

function targetFlags(target: string): string[] {
  if (target === 'uncommitted') return ['--uncommitted'];
  if (/^[0-9a-f]{7,40}$/i.test(target)) return ['--commit', target];
  if (target === 'HEAD') return ['--commit', 'HEAD'];
  return ['--base', target];
}

/**
 * Build the `codex review` arg array. The model is set with `-c model=` (the
 * subcommand exposes no `-m`); effort via `model_reasoning_effort`. The prompt
 * (custom instructions) is delivered on stdin by the caller via the `-` sentinel.
 */
export function buildReviewArgs(inv: ReviewInvocation): string[] {
  return [
    'review',
    ...targetFlags(inv.target),
    '-c',
    `model="${inv.model}"`,
    '-c',
    `model_reasoning_effort="${inv.effort}"`,
    '-c',
    'approval_policy="never"',
    '-',
  ];
}
```

Design (mirrors `Controller.delegate` but simpler):

- Collaborators (injected): `runReview(args, {cwd,input,timeoutMs,auth})` for the
  native path (returns `{exitCode, stdout, stderr, timedOut}`); the existing
  `Executor` for the read-only exec path; `MultiAuth`; `Ledger`; `now`.
- `review(spec, policy)`:
  1. `resolved = resolveReview(policy, spec.reviewType)`; if null → `hand_back`

  2. Fallback ladder over `resolved.chain` using `classifyFailure` + `nextAction`

       model, effort, sandboxLevel:'read-only', auth, timeoutMs})`, findings =
       `result.report` (last-message).
  4. exit 0 → `{status:'done', findings, model, effort}` + ledger `rung:'review'`.

     A one-paragraph "when to reach for each."
  2. **Decision table** (task → mode → subcommand → model → effort → cost):

     | I want to… | mode | subcommand | model | effort |
     | --- | --- | --- | --- | --- |
     | rename / apply-diff / boilerplate | execute | `delegate` (mechanical) | luna | medium |
     | build a feature / guided refactor | execute | `delegate` (implementation) | terra | medium |
     | crack a hard, specifiable bug | execute | `delegate` (hard) | sol | high |
     | sanity-check a routine diff | judge | `review` | terra | high |
     | critical / security-sensitive diff | judge | `review` (override) | sol | high |
     | audit an area for issues | judge | `audit` | sol | xhigh |
     | second-opinion a plan before work | judge | `plan-review` | sol | high |

  3. **The Sol clause** — "sol can exceed your precision on judgment. Treat a
     review as a genuine second opinion, but VERIFY each finding against the code
     before acting; never apply blindly, never dismiss blindly. You stay
     accountable for what lands." (Cite the empirical Phase-A example: a terra
     code-review caught a real changelog/impl mismatch.)
  4. **Cost discipline (autonomous, no hard gate)** — sol is most expensive; use
     it where correctness pays (hard execution; audit/plan-review; critical
     code-review). Routine diffs → terra.
  5. **ReviewSpec reference** — fields table (reviewId, repoPath, reviewType,
     target semantics per type, focus, model/effort overrides, auth) + a JSON
     example per review type.
  6. **Planning hook** — when a plan step says "delegate to Codex here," Claude
     already knows: execute or judge, which model/effort, and what it will do with
     the result (for a review: read → verify top findings → decide).
  7. Keep + extend **"When NOT to delegate"**: also "don't request a review you
     won't read and verify."
